# Retrospective Template

> Use this template for sprint retrospectives. Includes facilitation guide.

## Template

```markdown
# Sprint [N] Retrospective: [Team Name]

**Date:** [YYYY-MM-DD]
**Facilitator:** [Name]
**Attendees:** [Team members]
**Sprint Period:** [Start] — [End]

## Sprint Summary

| Metric | Value |
|--------|-------|
| **Sprint Goal** | [What we set out to achieve] |
| **Goal Achieved?** | Yes / Partially / No |
| **Story Points Committed** | [X] |
| **Story Points Delivered** | [X] |
| **Completion Rate** | [X%] |
| **Incidents** | [Number] |
| **Carry-over Items** | [Number] |

## Retrospective Format: [Start / Stop / Continue]

### 🟢 Start (What should we begin doing?)

| Idea | Votes | Action Item | Owner | Due |
|------|-------|-------------|-------|-----|
| [...] | [X] | [Specific action] | [Name] | [Date] |

### 🔴 Stop (What should we stop doing?)

| Idea | Votes | Action Item | Owner | Due |
|------|-------|-------------|-------|-----|
| [...] | [X] | [Specific action] | [Name] | [Date] |

### 🔄 Continue (What should we keep doing?)

| Idea | Votes | Notes |
|------|-------|-------|
| [...] | [X] | [Why this is working] |

## Team Health Check

| Question | Score (1-5) | Notes |
|----------|-------------|-------|
| How satisfied are you with this sprint? | [X]/5 | [...] |
| How sustainable was our pace? | [X]/5 | [...] |
| How well did we collaborate? | [X]/5 | [...] |
| How clear were our goals? | [X]/5 | [...] |
| How supported did you feel? | [X]/5 | [...] |

## Action Items from Previous Retro

| Action | Owner | Status | Notes |
|--------|-------|--------|-------|
| [Previous action] | [Name] | ✅ Done / ⏳ In Progress / ❌ Dropped | [...] |

## Key Takeaways

[2-3 sentences summarizing the most important insights from this retro.]

## Next Retro

**Date:** [Date of next retrospective]
**Facilitator:** [Name — rotate each sprint]
**Format:** [Start/Stop/Continue, Glad/Sad/Mad, 4Ls, etc.]
```

## Facilitation Guide

### Before the Retro (5 minutes prep)

1. **Review sprint data:** Completion rate, incidents, carry-over items
2. **Prepare the board:** Set up the retro format in Miro, physical board, or Confluence
3. **Check previous actions:** What actions from the last retro are complete?

### During the Retro (45 minutes)

#### 1. Check-in (5 minutes)
```
"Let's start with a quick check-in. One word: how are you feeling
about this sprint?"

Go around the room. No judgment, just listening.
```

#### 2. Review Sprint Data (5 minutes)
```
"Here's the data from this sprint:
- We committed to 26 points, delivered 22 (85%)
- We had 1 incident (the LLM API timeout)
- 2 items carried over

Let's look at the trends:
[Show velocity chart, incident trend, etc.]
```

#### 3. Brainstorm (10 minutes) — Silent Writing
```
"Take 5 minutes to silently write down your thoughts on the board.
Think about:
- What went well?
- What didn't go well?
- What surprised you?
- What should we change?

Write one idea per sticky note. Be specific."
```

#### 4. Group and Vote (5 minutes)
```
"Let's group similar ideas together. Now, each person gets 3 votes.
Vote for the items you think are most important to discuss."
```

#### 5. Discuss Top Items (15 minutes)
```
"Let's discuss the top 3 voted items. For each:
- What happened?
- Why did it happen?
- What can we do about it?

Focus on actions, not complaints."
```

#### 6. Commit to Actions (5 minutes)
```
"Let's commit to 1-2 specific actions for next sprint. Each action
needs an owner and a due date. Be realistic — we'll review these
at the next retro."
```

### After the Retro

1. **Publish the retro notes** within 24 hours
2. **Add action items** to the sprint backlog or team tracker
3. **Share trends** with engineering manager (aggregate, not individual)
4. **Schedule next retro** and rotate the facilitator

## Alternative Retro Formats

### Glad / Sad / Mad

```
GLAD: What made you happy this sprint?
SAD: What disappointed you?
MAD: What frustrated you?
```

Best for: Teams that need to process emotions after a tough sprint.

### 4Ls

```
LIKED: What did you like?
LEARNED: What did you learn?
LACKED: What was missing?
LONGED FOR: What do you wish we had?
```

Best for: Teams focused on growth and improvement.

### Sailboat

```
🏝️ Island (Goal): Where are we trying to go?
⛵ Sail (Wind): What's pushing us forward?
⚓ Anchor: What's holding us back?
🪨 Rocks: What risks are ahead?
```

Best for: Teams working toward a specific goal or launch.

### Mad/Sad/Glad + Action Matrix

```
After collecting Glad/Sad/Mad items, plot them on a matrix:
- Y-axis: Impact (High/Low)
- X-axis: Control (In our control / Out of our control)

Focus discussion on High Impact + In Our Control items.
```

Best for: Focusing on actionable items, not venting about things the team can't change.

## Retro Anti-Patterns

| Anti-Pattern | What It Looks Like | Fix |
|--------------|-------------------|-----|
| **Complaint session** | All negative, no actions | Focus on "what can we change?" not "what's wrong?" |
| **Action graveyard** | Actions from last retro never done | Limit to 1-2 actions per retro; review them first |
| **Manager-dominated** | Manager talks 80% of the time | Facilitator ensures equal participation |
| **No psychological safety** | People don't share honestly | Start with a positive check-in; model vulnerability |
| **Same format always** | Retro feels rote and boring | Rotate formats; try new activities |
| **No follow-through** | Retro happens, nothing changes | Actions are tracked and reviewed each sprint |
