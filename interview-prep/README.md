# Interview Preparation Guide

## Overview

This guide covers the complete interview preparation process for GenAI engineering roles in banking. It is organized by interview stage and includes resources for technical, behavioral, and domain-specific preparation.

## Typical Interview Process

| Stage | Format | Duration | Focus |
|---|---|---|---|
| **1. Recruiter Screen** | Phone call | 30 min | Background, motivation, compensation |
| **2. Technical Phone Screen** | Video call | 45-60 min | Coding or system design fundamentals |
| **3. Technical Assessment** | Take-home or live coding | 2-4 hours | Coding, SQL, or system design |
| **4. Onsite (4-5 rounds)** | Back-to-back interviews | 4-5 hours | Deep technical, system design, behavioral, leadership |
| **5. Final Round** | With VP/Director | 30-45 min | Strategic thinking, cultural fit |

## Preparation Timeline

### 8 Weeks Before
- [ ] Review all topics in this academy
- [ ] Start practicing coding exercises (1-2 per day)
- [ ] Begin building STAR stories (see [star-stories.md](star-stories.md))
- [ ] Research the bank's GenAI initiatives

### 6 Weeks Before
- [ ] Complete system design study (see [../system-design/](../system-design/))
- [ ] Practice SQL exercises (see [sql-exercises.md](sql-exercises.md))
- [ ] Review RAG concepts (see [../rag-and-search/](../rag-and-search/))
- [ ] Start mock interviews with a peer

### 4 Weeks Before
- [ ] Complete all coding exercises (see [coding-exercises.md](coding-exercises.md))
- [ ] Practice system design whiteboard problems
- [ ] Review banking domain knowledge
- [ ] Refine STAR stories

### 2 Weeks Before
- [ ] Full mock interviews (simulating real conditions)
- [ ] Review weak areas identified in mocks
- [ ] Prepare questions to ask interviewers
- [ ] Research interviewers on LinkedIn

### 1 Week Before
- [ ] Light review of key concepts
- [ ] Rest and mental preparation
- [ ] Prepare logistics (outfit, tech setup for video)

## Interview Evaluation Criteria

### Technical Excellence (35%)
- Coding ability: clean, efficient, well-tested code
- System design: scalable, maintainable, well-reasoned architecture
- Domain knowledge: understands GenAI, RAG, banking technology
- Problem-solving: approaches problems methodically

### Communication (25%)
- Explains thinking clearly
- Asks clarifying questions
- Handles ambiguity well
- Collaborative in problem-solving

### Banking Context (20%)
- Understands banking constraints (security, compliance, audit)
- Knows relevant regulations at a high level
- Appreciates the difference between consumer tech and banking tech

### Cultural Fit (20%)
- Collaborative, not ego-driven
- Handles feedback well
- Shows genuine interest in the role
- Demonstrates growth mindset

## How to Use This Guide

1. **Start with the README** in each subfolder to understand scope
2. **Practice, don't just read**: Write out answers, don't just think them
3. **Mock interviews are critical**: At least 5 before the real thing
4. **Track your weak areas**: Focus study time where you need it most
5. **Bank first, AI second**: Always frame answers in banking context
